Welcome to Strife

Introduction


A Fast and Lightweight PHP MVC Framework.
Developed for any type of web applications. Composed of handy and powerful set of tools for faster and secure development of robust applications.
Strife Framework’s first release was November 10, 2015 as a stable release, its aim is simplicity, speed, and easy to understand coding structure.
It is compatible with PHP 5.6 and later versions of PHP, It works perfect with MySQL and supports Apache’s mod_rewrite plugin.

Strife features:

Smart Routing
  • Uses pretty URLs and Smart Routing class.
  • Provides greater flexibility in handling forms and CRUD.
Query Builder
  • Has beautiful coding structure for the benefit of the developer.
  • "Greater Readability = Greater Scalability" chaining queries together to form a beautifully written code.
Multi-Database Support
  • Can easily manage multiple databases of your choice
  • Includes Migration manager, Seeders, Backup/Restore directly accessible using Yamato Command Line Interface
Form Builder
  • Powerful class for Automated forms that handles form errors and auto-populates inputs for you
HTTP Request Handlers
  • Handles form validations, auto-populates, and handles file uploads flawlessly.
Migration Manager
  • Build your database using Migration Manager in few taps, gives you full access to migrating your database for production using YamatoCLI
Encryption/Hashing
  • Handles one-way data hashing algorithm for strong passwords as well as Encryption and Tokens for password management
Command Line Interface
  • A Powerful CLI Application that packs a punch built for Strife Framework
Template Engine
  • Your code is your masterpiece, A beautiful, fast and easy to use Template Engine is your tool to make it happen.
Page Caching
  • Pages and Templates are cached automatically and that makes every pages load faster.
Helper Functions
  • Includes bunch of helper functions at your fingertips.

Server Requirements

  • PHP version 5.6 or newer is required to be installed.
  • MySQL (5.1+) via the PDO driver
  • Composer via https://getcomposer.org
  • Apache’s mod_rewrite module (Required for Virtual Host)

Getting Started

Manual Installation

NOTE: Using this framework assumes you already have a basic knowledge of core PHP as well as familiarity with Object-Oriented Programming(OOP), you will learn eventually about how a Model-View-Controller works within this documentation.
  • Download the latest release on Github extract it to public_html or project root.
  • You can also clone master branch:
git clone https://github.com/strifejeyz/framework

All the steps given should give you the Strife landing page. At that point, you won't be able to utilize yet the routing engine since Strife uses mod_rewrite module in Apache for URL routing and we NEED to setup the Virtual Host in your local computer so we can make it look like for example this: http://myapp.dev/

Composer Installation

Composer is a very useful package manager to install third party plugins like Image Intervention, Faker, and hundreds of thousands of open-source packages available to extend your application. It is mainly used by Strife Framework for autoloading classes as it’s easier to install project dependencies and plugins.

It is recommended to install Composer if you are going to use the built-in Command Line Interface in this framework. Refer to getcomposer.org for a complete tutorial on how to install it with Linux/Unix/Mac and for Windows.


Running your first app

If you are testing it in your live server, make sure index.php and other project files are in the same directory
inside your public_html (project root) and you can run it using your base url e.g. http://example.com
If you are on a local environment (which most of the time you will be), you can test it like so http://localhost/myapp
assuming myapp is the directory where the project files are in or you can cd to your project root and run php serve index.php

Well, it’s not over yet. Since Strife Framework uses smart URL routing, we NEED to setup the Virtual Host in your local computer so we can make it look like for example: http://myapp.dev/


Setting up Virtual Host

Virtual Host Installation for Windows

1. Open and run Command Prompt as Administrator

2. Issue the command echo 127.0.0.1 myapp.dev >> C:/Windows/System32/drivers/etc/hosts where myapp.dev for example is your custom server name. You can pick whatever you like.

3. using any text editor or notepad, open the httpd-vhosts.conf file.
for xampp, usually in c:/xampp/apache/conf/extra/httpd-vhosts.conf
for wamp, usually in C:/wamp/bin/apache/AapcheXXX/conf/extra/httpd-vhosts.conf

4. Now, add this block of code: 
<VirtualHost *:80>
    DocumentRoot "C:/Xampp/htdocs/myapp/"
    ServerName myapp.dev
</VirtualHost>

5. Save, and restart your server.
6. Now, visit http://myapp.dev/
7. If you don’t wanna do all this stuff, I have created a shell script (for windows) check out this vHost Configurator


Virtual Host Installation for Linux/Mac

  1. Next, we’ll add our custom domain sudo echo 127.0.0.5 myapp.dev >> /etc/hosts
  2. We need to enable vhosts first, sudo gedit /opt/lampp/etc/httpd.conf
  3. Now locate yourself in (about) the line 487 where you probably will find the following lines: 
    # Virtual hosts
#Include etc/extra/httpd-vhosts.conf
    As you can see, the Include statement that includes the httpd-vhosts.conf file is commented. Proceed to modify the line uncommenting that line: 
    # Virtual hosts
Include etc/extra/httpd-vhosts.conf
  4. You need to add the virtual host in /opt/lampp/etc/extra/httpd-vhosts.conf Use your favorite editor to edit that file.
  5. The port that we will use will be 127.0.0.5 
        <VirtualHost 127.0.0.5:80>
    DocumentRoot "/opt/lampp/htdocs/myapp"
    DirectoryIndex index.php

    <Directory "/opt/lampp/htdocs/myapp">
    Options All
    AllowOverride All
    Require all granted
        </Directory>
</VirtualHost>
6. Finally, restart your server. sudo /opt/lampp/lampp restart and you should get this when you go to http://myapp.dev/
